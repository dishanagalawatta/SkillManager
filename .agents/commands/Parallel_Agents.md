Execution Instructions:
For the primary task provided, autonomously map the relevant codebase context and orchestrate the execution using parallel agents. You must first discover the necessary files, break down the primary task into independent sub-tasks, assign them to virtual agents, and synthesize their final outputs.

Thinking Process:
Step 1 (Auto-Discovery): Analyze the primary task and scan the codebase/environment to autonomously identify the specific files, routes, components, and architectural dependencies required.
Step 2 (Task Breakdown): Divide the primary task into discrete, independent components using the discovered files. 
Step 3 (Delegation): Define specific, isolated scopes for each parallel agent to prevent merge conflicts.
Step 4 (Execution): Execute the implementation for each agent's scope.
Step 5 (Synthesis): Review the combined output for integration issues.

Constraints:
- Do not ask for file paths; autonomously identify the target files derived from the primary task and codebase structure.
- Sub-tasks assigned to agents must be genuinely parallelizable and non-overlapping.
- Do not modify files outside of the explicitly assigned scope for each agent.
- Include necessary imports and dependencies in the final output.
- If a dependency blocks parallel execution or a critical file cannot be found, flag it immediately before generating code.

Format:
Return the response strictly in the following Markdown structure:

### 1. Auto-Discovery Report
* **Target Files Identified:** [List the exact file paths discovered that need modification or creation]
* **Contextual Overview:** [Brief summary of how the primary task integrates with the discovered architecture]

### 2. Parallel Task Breakdown
* **Agent A ([Focus Area, e.g., Backend]):** [Target Files] - [Brief description of what it will build]
* **Agent B ([Focus Area, e.g., Frontend]):** [Target Files] - [Brief description of what it will build]
* **[Add additional agents as needed]**

### 3. Agent Execution Outputs
[Provide the actual code/implementation generated by each agent, separated by clear headers]

### 4. Integration Summary
[Briefly explain how the parallel outputs connect together and list any manual steps required to merge them]
