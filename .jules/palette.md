## 2025-02-12 - Proper Key event propagation handling
**Learning:** In QtQuick components designed to handle specific keys while still allowing unhandled keys (like Up/Down arrows for navigation) to pass through to native items, explicitly setting `event.accepted = false` for unhandled keys and `event.accepted = true` for handled keys is required. `Qt.Key_Escape` needs to be manually handled in custom popups to properly emulate native control behavior.
**Action:** When auditing or implementing `Keys.onPressed` in interactive UI controls, ensure both accepted and unhandled keyboard events have their `event.accepted` flags explicitly managed.
