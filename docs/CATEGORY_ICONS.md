# 🗺️ Skill Manager: Category Icon Specification

This document defines the visual identity and category-to-emoji mapping system for the Skill Manager. It serves as the single source of truth for both the UI implementation and the sourcing of 3D assets.

---

## 🎨 Visual Identity Standard

### 1. Emoji-Based Categorization
Every category in the Skill Manager is assigned a unique, semantically relevant emoji. This ensures immediate visual recognition even without custom assets.

### 2. 3D Isometric Assets
For high-fidelity views, the application supports **Flat 3D Isometric** icons.
- **Style**: Modern, sleek, glassmorphism, or flat 3D.
- **Specification**: Follow the [Icon Design Specification](file:///c:/Users/DIKKA/Documents/01-Projects/20-AiSupportTools/SkillManager/assets/docs/icon_specification.md).

## 🔧 Implementation Details

### Emoji Resolution
The application uses the `AppController.getCategoryEmoji(category_name)` method to resolve these icons. The logic follows a three-step matching process:
1. **Exact Match**: Direct mapping for standard categories.
2. **Case-Insensitive Match**: Resilience against naming variations.
3. **Substring Match**: Supports aliases (e.g., "Web Dev" maps to "Web Development").

### Maintenance
When adding a new category to the library:
1. Add the category and its chosen emoji to the tables above.
2. Update the `mapping` dictionary in `src/skill_manager/app.py`.
3. (Optional) Source a 3D SVG icon following the keywords provided.
