# Workspace Housekeeping

> Rules for keeping the SkillManager workspace clean. This document
> is the living reference for what gets cleaned, when, and what is
> permanently excluded.

## Cleanup Triggers

Run a full cleanup before any of the following:

- **Pre-release.** After `python-semantic-release` bumps the version.
- **Monthly.** First Monday of each month (automated via CI or manual).
- **Post-merge.** After a large feature branch merges to `develop`.

## Targets

### Always safe to remove (gitignored, regenerable)

| Path | Regenerated by |
|------|----------------|
| `build/` | `scripts/build_app.py` / PyInstaller |
| `dist/` | Release pipeline |
| `__pycache__/` (all subtrees) | `uv sync` / Python import |
| `.pytest_cache/` | `uv run pytest` |
| `.ruff_cache/` | `uv run ruff check` |
| `.mypy_cache/` | `uv run mypy` |
| `.qmlcache/`, `.qmltypes/`, `*.qmlc`, `*.jsc` | PySide6 / Qt |
| `.coverage`, `.coverage.*` | `uv run pytest --cov` |
| `coverage.xml`, `coverage.json` | `uv run pytest --cov` |
| `src/skill_manager.egg-info/` | `uv sync` |
| `tuf_keys/`, `tuf_repo/`, `.tufup-repo-config` | Deprecated (ADR-0010) |
| `.opencode/` | Local agent tooling config (ADR-0016) |

### Conditional removal (check first)

| Path | Condition |
|------|-----------|
| `conductor/_archive/` | Only if all contained tracks are confirmed released |
| `data/` | Runtime state; only remove if app is not running |
| `src/data/` | Runtime state inside source tree; should not exist in production |

### Manual review

| Path | Action |
|------|--------|
| `conductor/` root `*.md` plans | Archive to `conductor/_archive/<date>/` if idle ≥ 30 days (ADR-0015) |
| `*.py` at repo root | One-off migration scripts → archive to `conductor/_archive/<date>/` or move to `scripts/` (ADR-0018) |
| `.env` | Never commit. Verify `.env.example` is up to date. |

## Exclusions (never clean)

These files and directories are **permanently excluded** from cleanup:

- `TODO.md` — user-managed task list
- `.agents/commands/**` — user-managed agent commands
- `.agents/skills/**` — installed agent skills
- `image/TODO` — packaging reference
- `src/` source code — only caches are cleaned
- `tests/` source code — only caches are cleaned
- `docs/` — documentation is never cleaned
- `conductor/tracks/*/` — active and completed tracks are preserved

## Verification After Cleanup

After any cleanup, run the quality gates:

```bash
export QML_DISABLE_DISK_CACHE=1
uv run ruff check src tests
uv run ruff format --check src tests
uv run pytest
python run_tests.py
```

All gates must pass before committing. Also verify `git status` shows
no unexpected untracked files (caches reappearing indicate missing
`.gitignore` rules).
